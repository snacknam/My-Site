Exemble  —  사내 문서 기반 LLM 워크플로우 환경 설계하기

B2B 프로덕트 디자인 · 2025.10 - 2026.01
Exemble은 기업 내 데이터와 자체 모델로 구성된 에이전트들을 워크플로우로 구성하여, 채팅을 통해 다양한 업무를 수행하는 온프레미스 AI 플랫폼입니다. 사용자용 채팅 · 워크플로우부터 관리자용 운영 · 권한 · 모니터링까지 핵심 경험들을 설계하고, 초기 서비스 기획부터 팀의 단독 디자이너로서 프로젝트를 주도했습니다.

사내 데이터를 활용하는 채팅 워크플로우

Exemble은 일반적인 LLM 채팅 서비스와 달리, 기업 내부의 데이터와 외부 API, 그리고 이를 구성하는 모델과 권한 체계를 함께 다뤄야 하는 제품입니다. 사용자는 일반적인 질문을 넘어 사내 문서를 바탕으로 답변을 생성하거나 특정 워크플로우를 실행해 업무 결과물을 만들어 낼 수 있고, 관리자 입장에서는 이 과정을 운영하고 통제할 수 있어야 했습니다.
채팅 워크플로우 시작

사용자 화면과 관리자 화면의 구분

서비스는 크게 채팅 · 워크플로우를 실행하는 사용자 화면과 이에 필요한 다양한 요소들을 관리하고 운영하는 관리자 화면으로 나뉩니다. 각 화면의 목적에 따라 레이아웃과 화면의 복잡도 등을 다르게 하여 사용자 화면은 매끄러운 작업 흐름을, 관리자 화면은 리소스를 손쉽게 관리할 수 있도록 설계했습니다.
관리자 / 사용자 메뉴 구조

복잡한 작업을 직접 구성할 수 있는 작업 공간

워크플로우는 서비스의 핵심 경험으로, 사용자가 에이전트 · API 도구 · 조건 분기 등을 조합해 실제 업무를 직접 구성할 수 있습니다. 각 노드들을 연결해 다양한 업무 흐름을 시각화 할 수 있고, 단순한 다이어그램이 아닌 실행 가능한 구조로서 상태 표현과 테스트/디버깅이 가능하도록 설계했습니다.
업무 흐름의 시각화

노드 연결을 넘어 실제 편집 가능한 빌더

초기 워크플로우는 에이전트와 도구를 다른 메뉴에서 생성해야 했고, 작은 핸들을 직접 연결해야만 하는 등 실제 편집 과정에서 조작 비용이 높았습니다. 이후 화면 전체를 캔버스로 활용할 수 있도록 구조를 바꾸고, 워크플로우 안에서 생성·연결·수정이 이어지도록 개선하면서 자동 정렬, 화면 맞춤, 단축키, 복수 분기 같은 편의 기능도 함께 강화했습니다.
워크플로우 내 에이전트 생성

질문에서 실행까지 이어지는 채팅 경험

사용자 채팅은 단순한 질의응답을 넘어서, 문서 업로드와 워크플로우 실행을 통해 실제 업무를 처리하는 인터페이스여야 했습니다. 일반 채팅과 워크플로우 실행의 흐름을 자연스럽게 연결하면서도, 입력과 결과 확인의 부담을 줄일 수 있는 구조를 고민했습니다.

근거를 확인하고 결과를 검토하는 AI 채팅

사내 데이터를 바탕으로 답변을 생성하는 서비스인 만큼, 사용자는 결과만 보는 것이 아니라 어떤 자료를 근거로 응답이 만들어졌는지도 함께 확인할 수 있어야 했습니다. 채팅 화면에서 참조한 문서를 우측 패널로 확인할 수 있도록 구성해, AI 결과를 검토 가능한 경험으로 설계했습니다.

데이터·모델·에이전트의 관계를 운영 가능한 구조로 정리하기

Exemble은 데이터 업로드와 벡터화, 모델 관리와 파인튜닝, 에이전트 생성과 성능 평가처럼 기술적으로 연결된 개념을 함께 다루는 서비스였습니다. 복잡한 기술 개념을 그대로 드러내기보다, 각 기능의 역할이 분명하고 운영 흐름이 자연스럽게 이어지도록 정보 구조와 화면 관계를 정리했습니다.

성능 평가와 모니터링으로 AI를 운영 가능한 서비스로 만들기

AI 서비스는 결과를 생성하는 것만으로 끝나지 않고, 응답 품질과 시스템 상태를 지속적으로 확인하고 관리할 수 있어야 합니다. 에이전트의 평가 데이터와 실행 결과, 리소스 사용량과 오류율 같은 운영 지표를 대시보드와 로그 화면으로 설계해 실제 서비스 운영이 가능한 구조를 만들었습니다.

AI 경험을 이끌어주는 인터랙션과 애니메이션

Exemble의 인터랙션은 단순한 시각 효과보다, AI가 생각하고 실행하고 연결되는 상태를 사용자에게 이해시키는 역할에 더 가까웠습니다. 채팅 생성 중 애니메이션, 스크롤 이동, 워크플로우 연결 흐름, 화면 전환 방식 등을 통해 복잡한 AI 경험이 더 자연스럽게 느껴지도록 설계했습니다.

하나의 AI 기능이 아니라 하나의 운영 체계를 설계하다

Exemble은 채팅 화면 하나를 만드는 프로젝트가 아니라, 사용자 실행 경험과 관리자 운영 경험이 함께 맞물리는 AI 서비스 전체를 설계하는 작업이었습니다. 이 프로젝트를 통해 복잡한 기술 구조를 제품 경험으로 번역하고, 빠른 일정 속에서도 서비스의 골격을 세우는 방식을 구체적으로 경험할 수 있었습니다.
